# Stop Hook: Append-to-Markdown Conversation Logger for Cross-Session Grep Memory

## What Is This
The Claude Code `Stop` hook fires once per turn after Claude finishes responding. Wiring it to a shell script that appends timestamped entries to a global markdown log builds a durable, searchable history of every conversation across all sessions — no vector DB, no embeddings, no plugins.

## Why It Matters
Claude has zero built-in cross-session memory — each session starts blank. This pattern gives Claude a grep-searchable corpus of everything you've ever discussed. You pull exact past context into any new session by grepping the log and pasting a snippet, bypassing context window size concerns. It's the cheapest possible long-term memory implementation.

## How To Use It

### 1. Create the hook script

```bash
mkdir -p ~/.claude/hooks
cat > ~/.claude/hooks/log-conversation.sh << 'EOF'
#!/bin/bash
# Called by Stop hook after every Claude response
# Stdin: JSON with session_id, cwd, permission_mode

LOG_FILE="$HOME/.claude/conversation-log.md"
TIMESTAMP=$(date -u +"%Y-%m-%dT%H:%M:%SZ")
INPUT=$(cat)

SESSION_ID=$(echo "$INPUT" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('session_id','unknown'))")
CWD=$(echo "$INPUT" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('cwd',''))")

echo "" >> "$LOG_FILE"
echo "## [$TIMESTAMP] session=$SESSION_ID" >> "$LOG_FILE"
echo "cwd: $CWD" >> "$LOG_FILE"
echo "---" >> "$LOG_FILE"
EOF
chmod +x ~/.claude/hooks/log-conversation.sh
```

### 2. Wire it in `~/.claude/settings.json`

```json
{
  "hooks": {
    "Stop": [
      {
        "type": "command",
        "command": "~/.claude/hooks/log-conversation.sh"
      }
    ]
  }
}
```

> **Note:** The `Stop` hook payload contains metadata only (`session_id`, `cwd`, `permission_mode`) — NOT the actual message text. To capture full conversation content, you need a `UserPromptSubmit` hook (captures user turn) plus a richer Stop hook, or use an `agent`-type hook with tool access to read transcripts.

### 3. Enhanced: capture both turns

```bash
# ~/.claude/hooks/log-user-prompt.sh — wire to UserPromptSubmit
INPUT=$(cat)
PROMPT=$(echo "$INPUT" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('prompt','')[:500])")
echo "### USER: $PROMPT" >> "$HOME/.claude/conversation-log.md"
```

```json
{
  "hooks": {
    "UserPromptSubmit": [
      { "type": "command", "command": "~/.claude/hooks/log-user-prompt.sh" }
    ],
    "Stop": [
      { "type": "command", "command": "~/.claude/hooks/log-conversation.sh" }
    ]
  }
}
```

### 4. Search the log from any session

```bash
# Find turns mentioning a topic
grep -i -B2 -A10 "redis caching" ~/.claude/conversation-log.md

# Pull a chunk to paste into a new session
grep -A20 "authentication flow" ~/.claude/conversation-log.md | head -40
```

### 5. Give Claude access to search its own memory

Add to `~/.claude/CLAUDE.md`:
```
If you need context from past conversations, run:
grep -i "TOPIC" ~/.claude/conversation-log.md
```
