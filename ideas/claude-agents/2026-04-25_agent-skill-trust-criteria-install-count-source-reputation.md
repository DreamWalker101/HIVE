# Agent Skill Trust Criteria — Install Count, Source Reputation, GitHub Stars

## What Is This
When skills from the open ecosystem are discovered via `npx skills find`, there are concrete trust signals to evaluate before installing. The Vercel-published `find-skills` skill codifies a 3-factor trust framework analogous to evaluating npm packages or browser extensions.

## Why It Matters
Agent skills run inside Claude Code's context window and shape how the agent reasons and acts. A malicious or low-quality skill can silently degrade output quality or inject bad instructions. Since skills.sh has 91K+ entries, filtering by trust signals is necessary.

## How To Use It

**3-factor trust check before installing any skill:**

| Signal | Threshold | Notes |
|---|---|---|
| Install count | ≥ 1,000 installs | < 100 installs = high skepticism warranted |
| Source reputation | Official orgs preferred | `vercel-labs`, `anthropics`, `microsoft` > unknown authors |
| GitHub stars | Check source repo | < 100 stars on source repo = treat with skepticism |

**Trusted official sources (safe defaults):**
- `vercel-labs/agent-skills`
- `anthropics/skills`
- `ComposioHQ/awesome-claude-skills`
- `microsoft/*`

**Red flags:**
- Author is unknown with no GitHub presence
- Skill has < 100 installs and was published recently
- Source repo has no stars or no activity

**If no trustworthy skill exists:**
```bash
# Create your own
npx skills init my-xyz-skill
```
