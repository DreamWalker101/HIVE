# Progressive Disclosure: Use CLAUDE.md as a Navigation Trunk, Not a Content Dump

## What Is This

Progressive disclosure is an information architecture pattern applied to LLM agent context management. Instead of loading all instructions, workflows, and tool documentation into CLAUDE.md upfront, you treat CLAUDE.md as a **navigation map** — a trunk that tells the agent *where to find* information, not the information itself. The agent then loads only the specific branches (files, skills, MCP docs) relevant to the current task.

## Why It Matters

Tool definitions in Claude Code already consume ~14–17K tokens per session before you type anything. A bloated CLAUDE.md compounds this. By using progressive disclosure, you give the agent access to a large surface area of workflows and tools without paying the token cost upfront — the agent only loads what it needs for each specific task. Anthropic's own engineering blog confirms this pattern as a core technique for effective agent context management.

## How To Use It

**Structure your CLAUDE.md as a decision tree, not a wiki:**

```markdown
# Project Agent Instructions

## Core Behavior
[minimal standing rules — 5-10 lines max]

## Available Workflows
When the user asks about X → load `.claude/workflows/X.md`
When working on frontend → read `.claude/context/frontend-conventions.md`
When debugging → follow `.claude/skills/debugging.md`
When creating a PR → load `.claude/skills/ship.md`

## MCP Tools Available
- `mcp-db`: database queries — see `.claude/tools/mcp-db-guide.md` for usage
- `mcp-browser`: web QA — see `.claude/tools/mcp-browser-guide.md`
```

**The agent will:**
1. Load CLAUDE.md automatically at session start (always in context)
2. Read only the specific workflow/skill file when the task matches
3. Never load unrelated context

**Directory layout that supports this:**
```
.claude/
  workflows/       # task-specific multi-step procedures
  context/         # domain knowledge (architecture, conventions)
  skills/          # reusable agent behaviors
  tools/           # MCP tool usage guides
CLAUDE.md          # trunk: points to all of the above
```

**Key rule:** If a piece of information is only needed for *some* tasks, it belongs in a branch file with a pointer from CLAUDE.md — not in CLAUDE.md itself.
