# Tiered Model Routing: Local Model for Bulk Tasks, Claude Opus for Precision

## What Is This
When running many agents simultaneously, route ~90% of tasks (classification, summarization, simple generation, triage) to a fast local model like Gemma 4 31B, and escalate only high-precision or complex reasoning tasks to Claude Opus. This creates a two-tier agent architecture where cost stays low but quality on critical tasks stays high.

## Why It Matters
Running even modest multi-agent pipelines exclusively on Claude Opus burns through API credits fast. Industry benchmarks confirm smart tiered routing cuts costs by 60–80%. For Ahmed's setup (RTX 3070 Ti, Gemma 4 31B or Qwen2.5-Coder-14B already installed), the local model tier is already free — only Opus calls cost money. This makes large-scale autonomous agent runs economically viable.

## How To Use It

**Step 1 — Classify task complexity at the router level:**
```python
def route_task(task: str) -> str:
    """Return 'local' or 'opus' based on task complexity."""
    simple_keywords = ["summarize", "classify", "extract", "format", "translate", "list"]
    if any(kw in task.lower() for kw in simple_keywords):
        return "local"
    return "opus"  # architecture decisions, debugging, code review, novel reasoning
```

**Step 2 — Wire both clients:**
```python
import anthropic
from openai import OpenAI  # Ollama uses OpenAI-compatible API

opus_client = anthropic.Anthropic()
local_client = OpenAI(base_url="http://localhost:11434/v1", api_key="ollama")

def run_agent_task(task: str) -> str:
    tier = route_task(task)
    if tier == "local":
        resp = local_client.chat.completions.create(
            model="qwen2.5-coder:14b",  # or gemma-4-31b when available in Ollama
            messages=[{"role": "user", "content": task}]
        )
        return resp.choices[0].message.content
    else:
        resp = opus_client.messages.create(
            model="claude-opus-4-6",
            max_tokens=4096,
            messages=[{"role": "user", "content": task}]
        )
        return resp.content[0].text
```

**Step 3 — Target split:** Aim for 80–90% local, 10–20% Opus. Log which tier each task hits and tune the router over time.
