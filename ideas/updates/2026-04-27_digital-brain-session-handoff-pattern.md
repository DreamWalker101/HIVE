# Digital Brain Session Handoff: Using External Knowledge Vaults to Bridge Claude Code Sessions

## What Is This

The "digital brain session handoff" pattern is an explicit workflow where you maintain an external markdown knowledge base (e.g., an Obsidian vault) and open each new Claude Code session by prompting Claude to read it. This bridges Claude's session amnesia without relying solely on CLAUDE.md's 200-line cap. The key is the *explicit prompt* at session start — not just having the files exist.

## Why It Matters

Claude Code's built-in memory is limited: CLAUDE.md tops out at ~200 lines before adherence degrades, and auto-memory is selective. For complex long-running projects with many tools, decisions, and sub-agent interactions, a richer external store lets Claude reconstruct meaningful context. It also works across *projects* — one vault, many repos.

## How To Use It

1. **Maintain a vault** — Obsidian (or any folder of `.md` files) with notes organized by project: decisions made, tools used, patterns discovered, gotchas hit.

2. **Add a session-start prompt** — Begin each Claude Code session with something like:
   ```
   Before we start, read ~/brain/projects/my-project/ and tell me what we were doing last time and what's pending.
   ```

3. **Have Claude write back to the vault** — At the end of a session or on demand:
   ```
   Summarize what we did today and append it to ~/brain/projects/my-project/log.md
   ```

4. **Leverage `~/.claude/projects/`** — Claude Code stores raw session JSON there. You can point Claude at this directly to reconstruct tool usage history:
   ```
   Read ~/.claude/projects/<project-hash>/ to see what tools were used in previous sessions.
   ```

5. **Complement, don't replace CLAUDE.md** — Keep CLAUDE.md for stable project instructions; use the external vault for dynamic session logs and evolving context.
