# Claude Code /loop Command: Native Recurring Scheduled Tasks

## What Is This
Claude Code has a built-in `/loop` slash command (bundled skill) that schedules a recurring prompt on a specified interval. It internally uses cron tooling (CronCreate/CronList/CronDelete) to re-run prompts automatically while the session is open. Requires Claude Code v2.1.72+.

## Why It Matters
This enables fully autonomous polling, monitoring, and iterative experiment loops without any external infrastructure. You can have Claude watch a deployment, run repeated checks, or drive an optimization loop — all from inside a single Claude Code session with one command.

## How To Use It

```bash
# Basic syntax
/loop <interval> <prompt>

# Examples
/loop 5m check if the deployment finished and report back
/loop 1h run one experiment iteration and store the result in results.json
/loop 30s poll the API endpoint and alert if latency exceeds 500ms
```

Underlying cron tools Claude can also call directly:
```
CronCreate  — schedule with standard 5-field cron expression
CronList    — list active scheduled tasks
CronDelete  — cancel a task by ID
```

**Key constraints to know:**
- Tasks are **session-scoped** — they die when you exit Claude Code
- Recurring tasks auto-expire after **3 days**
- For persistent cron jobs: use GitHub Actions, cloud scheduled tasks, or OS cron
