# Karpathy AutoResearch Pattern: Self-Optimizing Agent Loops for Any Measurable Business Metric

## What Is This
Andrej Karpathy's [AutoResearch](https://github.com/karpathy/autoresearch) is a ~630-line Python script that runs ML experiments in an autonomous loop: read research direction → propose a change → run experiment → evaluate metric → keep or revert → repeat. The key innovation is that the agent only needs two inputs: a **target metric** and **a constrained action space** (what it's allowed to modify). This pattern is fully transferable outside ML to any business function with a measurable output.

## Why It Matters
Most agent loops are one-shot. This pattern creates a *self-improving* loop that accumulates positive/negative learnings across runs, making each iteration smarter than the last. Applied with Claude Code's `/loop` feature, you can build an autonomous optimizer for ad copy CTR, cold email reply rate, landing page conversions, or any A/B-testable metric — with zero human involvement between runs.

## How To Use It

**Core architecture (3 components):**

```
1. METRIC SOURCE    — a number the agent can read (API, DB query, file)
2. ACTION SPACE     — explicit list of what the agent is allowed to change
3. MEMORY LOG       — append-only file of experiments + outcomes (positive/negative)
```

**Minimal implementation with Claude Code:**

```markdown
# research_direction.md
TARGET METRIC: cold_email_reply_rate (read from replies.json)
BASELINE: 4.2%
ACTION SPACE:
  - subject line wording
  - opening sentence
  - CTA phrasing
  - email length (±50 words)
CONSTRAINTS:
  - do NOT change sender name or company info
  - keep formal tone

EXPERIMENT LOG: experiments.md (append only — never delete entries)
```

```bash
# In Claude Code session
/loop 4h \
  "Read research_direction.md and experiments.md. Propose ONE change within the action space. \
   Apply it to email_template.txt. Log the proposed change to experiments.md with status=pending. \
   When metric data arrives, update status to win/loss and note the delta."
```

**Karpathy's original loop logic (adapted pseudocode):**
```python
while True:
    current_best = read_metric()
    propose_change()        # agent modifies artifact within constraints
    new_score = run_eval()  # measure the metric
    if new_score > current_best:
        keep_change()
        log("WIN: +{delta}")
    else:
        revert_change()
        log("LOSS: tried X, didn't work because Y")
```

**Notification hook (optional):**
Add a step that posts to Slack/Telegram webhook when a new best score is reached:
```bash
curl -X POST $SLACK_WEBHOOK -d '{"text": "New best: CTR improved to 6.1% (+0.4%)"}'
```
