# Output Token Compression via Explicit Behavioral Instructions in System Prompt

## What Is This
You can reduce Claude's output token count by ~50–65% on automation and agent tasks by giving it explicit behavioral rules in a system prompt or CLAUDE.md. The rules tell Claude to skip preamble, skip recaps, use structured output over prose, and answer directly. This is not about making Claude "dumber" — the information density stays the same, the filler goes away.

## Why It Matters
Output tokens cost **5× more** than input tokens on Anthropic's API. If you're running Claude in a loop — agent pipelines, repeated code generation, automation bots — verbose outputs compound fast. A 50–65% output reduction on those workloads translates directly to cost. For casual one-off queries the savings are negligible (the system prompt itself costs input tokens on every message, so net benefit only kicks in on output-heavy use).

## How To Use It
Drop these rules into your system prompt or CLAUDE.md for any automation / agent context:

```
RESPONSE RULES:
- No preamble ("Sure, I'd be happy to..." → just start)
- No recap ("So what I did was..." → skip it)
- No filler: "basically", "essentially", "actually", "just", "simply" → delete
- No narrating process ("I'm going to create a function that...") → just write it
- Structured output over prose for status/comparison (tables, bullet lists, diffs)
- One-liners for log entries and status updates
- Code over description for technical solutions
- Diffs over full rewrites when editing existing code
```

This is the high-signal part of the "Pulse Token Efficiency Compactor" document that circulated on Instagram. The rest of that doc is standard programming best practices (use arrow functions, list comprehensions, etc.) repackaged as AI advice — those are about making your *code* shorter, not about Claude's token usage specifically.

**For Claude Code specifically**, a CLAUDE.md with these rules in the project root persists them across all sessions automatically.

- ⚠️ The article's specific "60–70%" claim — directionally accurate but based on informal benchmarks, not statistically controlled studies; actual savings vary heavily by task type
- ❌ The article's BEFORE/AFTER token counts (e.g. "847 tokens → 210 tokens" for a JS function) are misleading — those count tokens *inside the code snippet*, not Claude's API response tokens. The real mechanism is behavioral instruction, not making your source code shorter.
- ⚠️ "Same output quality" — true for structured/agentic tasks; may degrade nuanced explanations or creative work where detail matters

## Caveat: When NOT to Use This
- Short, casual, one-off queries: the system prompt costs input tokens on every message, so you need enough output volume to break even
- Tasks requiring thorough explanation (teaching, debugging complex issues): brevity rules may cause Claude to skip useful context
- Best for: agent loops, repeated code gen, automation pipelines, structured data extraction
