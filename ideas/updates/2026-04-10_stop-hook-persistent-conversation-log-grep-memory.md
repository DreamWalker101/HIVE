# Stop Hook: Append-to-Markdown Conversation Logger for Cross-Session Grep Memory

## What Is This
The Claude Code `Stop` hook fires once per turn after Claude finishes responding. You can wire it to a shell script that appends a timestamped entry (user prompt + Claude summary) to a global markdown log file. Over time this builds a full searchable history of every conversation you've ever had across all sessions.

## Why It Matters
Claude has no built-in cross-session memory — each new session starts blank. This pattern creates a durable, human-readable, grep-searchable corpus of everything you've discussed without needing a vector DB, embeddings, or a plugin. You can pull exact past context into any new session by grepping the log and pasting a snippet, bypassing the context window size concern entirely.

## How To Use It

### 1. Create the hook script

```bash
mkdir -p ~/.claude/hooks
cat > ~/.claude/hooks/log-conversation.sh << 'EOF'
#!/bin/bash
# Called by Stop hook after every Claude response
# Stdin contains JSON with session_id, cwd, etc.

LOG_FILE="$HOME/.claude/conversation-log.md"
TIMESTAMP=$(date -u +"%Y-%m-%dT%H:%M:%SZ")

# Read the hook payload
INPUT=$(cat)
SESSION_ID=$(echo "$INPUT" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('session_id','unknown'))")

# Append a separator entry — Claude will have already written its response
# You can extend this to log actual turn content if you capture it via
# a PostToolUse hook on the Bash/Write tools, or via a prompt hook
echo "" >> "$LOG_FILE"
echo "## [$TIMESTAMP] session=$SESSION_ID" >> "$LOG_FILE"
echo "cwd: $(echo "$INPUT" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('cwd',''))")" >> "$LOG_FILE"
echo "---" >> "$LOG_FILE"
EOF
chmod +x ~/.claude/hooks/log-conversation.sh
```

### 2. Wire it in `~/.claude/settings.json`

```json
{
  "hooks": {
    "Stop": [
      {
        "type": "command",
        "command": "~/.claude/hooks/log-conversation.sh"
      }
    ]
  }
}
```

### 3. Enhanced version: capture full turn content via `prompt` hook type

Use a `UserPromptSubmit` hook to capture the user prompt and a `Stop` hook to append Claude's final response. For a richer log, use the `agent` hook type which has tool access and can read the actual session transcript.

### 4. Searching the log

```bash
# Find all turns mentioning a topic
grep -i -B2 -A5 "redis caching" ~/.claude/conversation-log.md

# Find by date range
grep "^\## \[2026-03" ~/.claude/conversation-log.md

# Pull a chunk to paste into a new session
grep -A20 "redis caching" ~/.claude/conversation-log.md | head -30
```

### 5. Tell Claude to search its own memory

In any new session, put this in CLAUDE.md or say it explicitly:
```
If you need context from past conversations, run:
grep -i "TOPIC" ~/.claude/conversation-log.md
```
