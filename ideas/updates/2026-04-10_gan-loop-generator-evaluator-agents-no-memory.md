# GAN Loop: Generator + Stateless Evaluator Agent Pattern for Quality Iteration

## What Is This
A two-agent adversarial loop inspired by Generative Adversarial Networks (GANs) from ML: Agent 01 (Generator) writes a draft, Agent 02 (Evaluator) scores it against a rubric with no memory of prior rounds, and the Generator revises. The loop runs until scores plateau — typically 2–3 rounds, going from 4/10 → 9/10 in the author's usage.

## Why It Matters
The critical non-obvious design is the **Evaluator has zero memory of previous rounds** — it re-reads the rubric fresh every pass. This prevents "grade inflation" drift where an evaluator goes easy on a draft just because it "improved" from the last version. Without this, evaluators reliably soften over iterations. With it, each pass is an independent cold assessment against the fixed rubric bar.

This is directly applicable to any Claude-based content pipeline: posts, code reviews, documentation, reports. Ahmed is already running multi-agent GSD loops — this pattern adds a structured quality gate before ship.

## How To Use It

### Agent 01 — The Generator
```
model: claude-sonnet-4-6       # use a proper versioned ID
tools: Edit, Write, Search
skills: [your domain skills]
mcp: [any needed integrations]

INPUT:  brief (what to write)
OUTPUT: draft.md
```

### Agent 02 — The Evaluator
```
model: claude-sonnet-4-6
inputs: rubric.md + draft.md
memory: NONE (critical — do NOT pass prior conversation)
```

**Evaluator rubric structure (example from post):**
```markdown
# rubric.md — 5 criteria, scale 1–10

HOOK STRENGTH     — does it open strong?
CLARITY           — is the main point surfaced early?
CONCRETENESS      — are there specific numbers/examples?
BRAND VOICE       — does it match the target tone?
CTA STRENGTH      — is the call-to-action specific?

VERDICT: PASS (≥8 avg) or REJECT · FIX & RETRY
```

### The Refine Loop
```
WRITE draft
  → SCORE against rubric (fresh context, no memory)
  → if REJECT: FIX (patch based on evaluator feedback)
  → RE-SCORE (fresh again)
  → repeat until score plateaus or PASS
```

Stop condition: score plateau OR hitting your max rounds (author uses 3).

### Implementing with Claude API
Each Evaluator call must be a **new conversation** — don't append evaluator output to the ongoing thread. Pass only: system prompt (containing rubric) + the current draft as user message. This enforces statelessness at the API level, not just by instruction.

### Cross-Lab Variant (Bonus)
Use Claude as Generator + OpenAI Codex (or any other model) as Evaluator for harder adversarial review. Different training data and RLHF means they'll catch different failure modes. Useful for code review or technically demanding content where blind spots differ by model.

## Implementation Notes
- The rubric lives in the **Evaluator's system prompt**, not passed as user message — this makes it a hard constraint rather than a suggestion
- Score plateau (not a fixed round count) is the cleaner stopping condition; 3 rounds is a practical ceiling
- The Generator's context should include the Evaluator's feedback from each rejected round, so it knows what to fix — the Evaluator's context should NOT include prior evaluation rounds
