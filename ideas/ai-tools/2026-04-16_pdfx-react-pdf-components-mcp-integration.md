# PDFx: React PDF Component Library with MCP Server Integration and AI-Ready Docs

## What Is This
PDFx (https://pdfx.akashpise.dev) is an MIT-licensed, open-source React PDF toolkit (v0.6.1) that provides copy-paste components, a theme builder, block layouts, and an AI-ready documentation layer. The notable detail: it ships with an **"MCP & Skills"** button in its navbar, indicating it has a Model Context Protocol server — meaning Claude and other MCP-compatible agents can call it directly to generate PDF documents.

## Why It Matters
PDF generation in AI agent workflows is typically painful — either server-side LaTeX, headless Chrome, or brittle HTML-to-PDF conversions. PDFx packages production-ready React components (invoices, service agreements, etc.) with a CLI init, theme system, and an MCP integration. This means an AI agent can invoke PDFx's MCP server to generate structured PDFs (invoices, reports, contracts) without custom implementation. The "AI-ready docs" framing also suggests the API surface is structured for LLM tool-calling.

## How To Use It
1. **Bootstrap a project:**
   ```bash
   npx pdfx-cli init
   ```
2. Browse copy-paste components at the Components section (invoices, service agreements, etc.)
3. Use the **Theme Builder** to set brand colors/typography once, propagated to all components
4. Check the **MCP & Skills** section in the navbar at https://pdfx.akashpise.dev to find the MCP server endpoint/config
5. Wire the MCP server into Claude Desktop or Claude Code to enable agent-driven PDF generation
