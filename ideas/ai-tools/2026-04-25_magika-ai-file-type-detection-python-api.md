# Magika: Google's AI File Type Detector with Python API for Pipeline Routing

## What Is This
Magika is Google's open-source AI tool for detecting file content types with ~99% accuracy. Unlike `file` or libmagic (which use static signatures), Magika uses a small deep learning model (~few MB) to classify 200+ content types — covering binary and text formats — in ~5ms per file on a single CPU. It exposes clean Python, Rust, JS, and CLI interfaces.

## Why It Matters
In AI pipelines that ingest arbitrary user files, knowing the true content type (not just the extension) is critical for routing to the right processor. Magika solves this with a programmatic API that returns structured output including label, MIME type, group, and confidence score — enabling conditional routing logic. The 5ms inference time (near-constant regardless of file size) makes it viable as a pre-processing step at scale.

## How To Use It

**Install:**
```bash
pip install magika
# or
pipx install magika
```

**Python API — three input modes:**
```python
from magika import Magika
m = Magika()

# From raw bytes (useful for in-memory data, API uploads)
res = m.identify_bytes(b'function log(msg) {console.log(msg);}')
print(res.output.label)        # → "javascript"
print(res.output.mime_type)    # → "application/javascript"
print(res.output.group)        # → "code"

# From file path
res = m.identify_path('./doc.ini')
print(res.output.label)        # → "ini"

# From stream (useful for streaming uploads)
with open('./doc.ini', 'rb') as f:
    res = m.identify_stream(f)
    print(res.output.label)    # → "ini"
```

**Routing pattern for AI pipelines:**
```python
from magika import Magika

m = Magika()
ROUTERS = {
    "pdf":        process_pdf,
    "docx":       process_word,
    "python":     process_code,
    "javascript": process_code,
    "csv":        process_tabular,
}

def route_file(path: str):
    res = m.identify_path(path)
    label = res.output.label
    score = res.dl.score  # confidence score 0-1

    if score < 0.7:
        return handle_uncertain(path)  # fallback for low confidence

    handler = ROUTERS.get(label, handle_unknown)
    return handler(path)
```

**Confidence/prediction modes via CLI:**
```bash
magika --high-confidence file.bin   # only trust high-confidence predictions
magika --best-guess file.bin        # always return best guess even if uncertain
magika -s file.bin                  # show prediction score alongside label
```

**Bulk processing:**
```bash
magika -r /uploads/          # recursive directory scan
magika file1 file2 file3     # thousands of files at once
magika --jsonl -r /uploads/  # structured output for pipeline ingestion
```

## Non-Obvious Details
- **File size doesn't matter for speed**: Magika only reads a limited content subset, so a 10GB file takes the same ~5ms as a 1KB file
- **Per-content-type thresholds**: The model uses different confidence thresholds per content type, not a single global threshold — reducing false positives on ambiguous types
- **Group field** (`res.output.group`) gives coarse categories like `code`, `document`, `text`, `binary` useful for broad routing before fine-grained label handling
- **Research paper**: Published at IEEE/ACM ICSE 2025 — not just a prototype, academically peer-reviewed
